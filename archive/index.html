CPU limit を超えると遅くなるだけなのに、メモリ limit だと Pod が落ちるのはなぜか？

CPU limit 超過は CFS スロットリングで待たされるだけだが、メモリ limit 超過は OOM Killer に強制終了される。requests/limits の役割の違いと QoS クラスによる eviction 優先順位を解説。

Pod・Deployment・Service、なぜ 3 つも要るのか？

Pod は使い捨ての最小実行単位、Deployment は ReplicaSet 経由で数と更新を管理し、Service は入れ替わる Pod 群を固定 IP の裏に隠す — 3 層に分かれているのは責務が違うからだ。
